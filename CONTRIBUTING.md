# Contributing to LeakPro

First off, thank you for taking the time to contribute! It’s people like you who make the open-source community such a great place.

You also represent that you are legally entitled to grant this license. If your employer has rights to intellectual property you create, you represent that you have received permission to make contributions on behalf of that employer.

## 2. Process for external contributors
1. **Search for Issues:** Before starting work, check the issue tracker to see if someone else is already working on the same thing.
2. **Open an Issue:** For major changes, please open an issue first to discuss what you would like to change.
3. **Fork the repository** by clicking "Fork" on the GitHub page
4. **Clone your fork, create a branch, and make your changes:**
`git clone https://github.com/<your-username>/LeakPro.git`
`git checkout -b my-feature-branch`
5. **Push to your fork:**
`git push origin my-feature-branch`
6. **Open a Pull Request** from your fork to `aidotse/LeakPro:main` and request a review — a maintainer will review and merge it

## 3. License Compliance Requirements
To maintain our Apache 2.0 compliance, every Pull Request must meet these criteria:
* **No Incompatible Licenses:** Do not introduce dependencies with GPL, AGPL, or "Non-Commercial" licenses.
* **Header Requirement:** New source files should include the standard Apache 2.0 boilerplate header.
* **Third-party attribution:** If you are adding code from another source, you must declare it so we can update the `NOTICE` file.
